# Redshift Emulator

A local Amazon Redshift emulator based on PostgreSQL 16. Implements Redshift-specific SQL functions, implicit casts, type domains, and mock system tables so that Redshift-targeting SQL can be tested locally without AWS credentials.

## Quick Start

```bash
docker compose up -d
```

Connect with any standard PostgreSQL client:

```bash
psql -h localhost -p 5439 -U admin -d dev
```

Check the startup smoke test passed:

```bash
docker compose logs redshift-emulator | grep "SMOKE TEST"
```

Expected output: `SMOKE TEST: ALL PASSED`

## Connection Details

| Parameter | Value       |
|-----------|-------------|
| Host      | localhost   |
| Port      | 5439        |
| Database  | dev         |
| Username  | admin       |
| Password  | admin       |

## DDL Preprocessor

Before loading a Redshift schema into the emulator, strip Redshift-only DDL clauses:

```bash
./scripts/ddl-preprocessor.sh my_schema.sql my_schema_pg.sql
psql -h localhost -p 5439 -U admin -d dev -f my_schema_pg.sql
```

The preprocessor removes: `ENCODE`, `DISTSTYLE`, `DISTKEY`, `SORTKEY` (including `COMPOUND`/`INTERLEAVED`), `AUTO REFRESH`, `BACKUP`, and converts `IDENTITY(seed,step)` to `GENERATED BY DEFAULT AS IDENTITY`.

## Implemented Functions

### Date/Time
`GETDATE`, `DATEADD`, `DATEDIFF`, `CONVERT_TIMEZONE` (2- and 3-arg), `ADD_MONTHS`, `MONTHS_BETWEEN`, `LAST_DAY`, `NEXT_DAY`, `DATE_CMP`, `DATE_CMP_TIMESTAMP`, `DATE_CMP_TIMESTAMPTZ`, `TIMESTAMP_CMP`, `TIMESTAMP_CMP_DATE`, `TIMESTAMP_CMP_TIMESTAMPTZ`, `TIMESTAMPTZ_CMP`, `INTERVAL_CMP`, `DATE_PART_YEAR`, `TIMEOFDAY`

### String
`LEN`, `CHARINDEX`, `REPLICATE`, `STRTOL`

### Conditional
`NVL`, `NVL2`, `DECODE` (3–11 args), `ZEROIFNULL`, `NULLIFZERO`

### Aggregate
`LISTAGG`, `MEDIAN`

### Array
`SPLIT_TO_ARRAY`, `GET_ARRAY_LENGTH`, `SUBARRAY`, `ARRAY_FLATTEN`

### Regex
`REGEXP_SUBSTR` (2- and 4-arg), `REGEXP_COUNT`, `REGEXP_INSTR`, `REGEXP_REPLACE` (4-arg)

### JSON
`JSON_EXTRACT_PATH_TEXT` (2–5 args), `JSON_EXTRACT_ARRAY_ELEMENT_TEXT`, `JSON_ARRAY_LENGTH`, `JSON_PARSE`, `JSON_SERIALIZE`, `CAN_JSON_PARSE`, `IS_VALID_JSON`, `IS_VALID_JSON_ARRAY`

### Math
`DEXP`, `DLOG1`, `DLOG10`

### Type Aliases
`SUPER` (→ JSONB), `VARBYTE` (→ BYTEA), `HLLSKETCH` (→ BYTEA)

## Known Limitations

- **`LISTAGG ... WITHIN GROUP (ORDER BY ...)`** syntax is not supported; rewrite as `STRING_AGG(col, ',' ORDER BY col)`.
- **Constraint enforcement**: PostgreSQL enforces PRIMARY KEY, UNIQUE, and FOREIGN KEY constraints. Redshift does not. Tests that insert duplicates will fail locally.
- **`COPY FROM S3` / `UNLOAD TO S3`** are not implemented.
- **`APPROXIMATE COUNT(DISTINCT col)`** is not supported as a syntax-level passthrough; use `COUNT(DISTINCT col)` instead.
- **False positives**: PostgreSQL 16 features not in Redshift (SAVEPOINTs, advanced index types, etc.) will work locally but fail on real Redshift.

## Rollback

```bash
docker compose down
```

```bash
docker compose down -v
```
